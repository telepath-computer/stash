# Remove `stash init` from CLI

## Why

`stash init` is a required step before `stash connect`, but a stash with no connection is just a folder — you can't do anything useful until you connect. Requiring `init` as a separate step adds friction to the most common flow (joining an existing stash) and an extra concept to explain.

## Behavior

`connect` auto-initializes the directory as a stash if `.stash/` doesn't exist. `init` is removed from the CLI.

**Before:**
```
stash init
stash connect github --repo user/repo
stash sync
```

**After:**
```
stash connect github --repo user/repo
stash sync
```

Existing stashes are unaffected — `connect` on an already-initialized directory skips the init step.

`Stash.init()` remains as a public static method on the class. It's still useful internally and in tests. It just isn't a CLI command anymore.
